# 圖片重新生成規格

## 範圍

`ASSET_MANIFEST.json` 是唯一檔案清單，共 248 張：

| 類別 | 數量 | 背景透明 |
|---|---:|---|
| UI | 76 | 是 |
| 玩家角色 | 80 | 是 |
| 怪物 | 36 | 是 |
| 背景 | 29 | 否 |
| 戰鬥特效 | 18 | 是 |
| Vertical Slice 圖示 | 4 | 是 |

manifest 的 `status` 由 `missing` 開始；每張通過驗收後才可改成 `ready`。

## 共通美術方向

- 題材：低魔、殘破邊境、冷鐵、厚布、舊皮革、灰燼與少量黃銅。
- 色彩：冷藍灰為底，餘燼紅表示傷害，清楚的藍色表示護盾，黃銅只用於焦點與獎勵。
- 光線：柔和方向光，角色與怪物輪廓優先；避免滿版霓虹、塑膠高光與廉價手遊光暈。
- 視角：角色與怪物統一 3/4 視角與腳底基線；玩家面右、怪物面左。
- 文字：圖片內禁止生成任何文字、數字、Logo 或浮水印。
- 透明圖：輸出真正 alpha，不可把白底、黑底或棋盤格畫進圖片。

## 生成順序

### 第一批：啟動與 UI（priority 1，共 105 張）

先完成首頁、地圖、五階戰鬥、事件／商店／營火背景，以及所有 UI 結構、意圖、藥水與天賦圖示。UI 圖示以 256×256 透明 PNG 生成，主體保留約 12% 安全邊界；九宮格框的中央與邊緣必須能被程式裁切拉伸。

### 第二批：角色與怪物（priority 2，共 116 張）

角色矩陣為 2 性別 × 4 種族 × 5 職業 × 站立／攻擊，共 80 張。怪物為 6 階 × 血量／攻擊／防禦三型 × 站立／攻擊，共 36 張；階級提升應增加威脅與裝備密度，但不可改變基線或裁切規則。

### 第三批：VFX 與專用圖示（priority 3，共 22 張）

18 張 VFX 必須在小尺寸仍能辨識方向與接觸點，效果本身置中且四周留透明空間。4 張 Vertical Slice 圖示為斬擊、防禦、藥水袋與獎勵餘燼，形狀語言需與戰鬥 HUD 一致。

## 背景構圖

- 正式視窗是 1180×720；主標題背景直接使用此比例。
- 戰鬥背景需保留左側玩家、右側敵人與底部 HUD 的低細節區。
- 事件、商店與營火背景中央要能承載文字面板，避免高對比物件穿過文字區。
- 冒險地圖的節點與路線由 UI 疊加，背景不要自行畫入不可互動的假節點或文字。

## 驗收門檻

- PNG 可由 Pillow 與 Arcade 解碼；實際尺寸與 manifest 一致。
- 透明素材為 RGBA，四角 alpha 接近 0，沒有大面積單色底。
- 角色與怪物可見內容不碰畫布邊緣，腳底透明留白一致。
- UI 圖示縮到 48–64 px 仍能一眼辨識，且不自帶第二層方框。
- HP 與 Shield 的形狀／色彩清楚不同。
- 相同批次不得出現畫風、鏡頭、光向與材質突然改變。
- 生成後只把驗收通過的成品放進 `assets/`；失敗稿與原始輸出留在專案外。

完成一個批次後執行 `python scripts/validate_assets.py --allow-missing`；全部完成後執行 `python scripts/validate_assets.py`，結果必須為 248 ready、0 missing、0 invalid。

## 角色動作與朝向規格

- 玩家角色採寫實感奇幻遊戲美術，保留自然成人比例與可信材質；所有站立與攻擊圖一律面向右側。
- 玩家 `pose: idle` 是基本站立；`pose: attack` 必須維持相同角色、服裝、武器與配色，向右完成攻擊動作。
- `warrior` 保留程式內職業代號，但美術形象統一改為狂戰士，不使用傳統盾牌坦克造型。
- 怪物採一致的寫實感奇幻遊戲美術，所有站立與攻擊圖一律面向左側。
- 怪物 `pose: idle` 是基本站立；`pose: attack` 必須維持相同怪物、裝備與配色，向左完成攻擊動作。
- 玩家與怪物的站立、攻擊圖必須共用地面基準線，完整保留全身、武器與肢體，不得裁切。
